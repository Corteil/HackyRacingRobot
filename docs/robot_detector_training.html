HackyRacingRobot — Robot Detector Training Guide

Train a custom YOLOv8n detector from race footage and deploy it to the Raspberry Pi AI HAT+ 2
Platform: Hailo-10H (40 TOPS)  ·  HailoRT 5.1.1  ·  YOLOv8n  ·  Windows + WSL2 pipeline

Overview

This guide trains a custom object detector that recognises other racing robots in real time using the Hailo-10H NPU on the Raspberry Pi AI HAT+ 2. The finished model (.hef file) slots into robot/robot_detector.py and feeds the camera thread in robot_daemon.py.

Step 1 — Enable WSL2 on Windows

WSL2 (Windows Subsystem for Linux) runs Ubuntu directly inside Windows. The Hailo compiler requires Linux x86-64 and runs natively here.

1a — Open PowerShell as Administrator

Right-click the Start button → Windows PowerShell (Admin) or Terminal (Admin).

1b — Install WSL2 with Ubuntu
wsl --install

This installs WSL2 and Ubuntu 24.04 LTS automatically. It will prompt you to restart Windows — do so.

1c — First-time Ubuntu setup

After restarting, Ubuntu opens automatically and asks for a username and password. Choose anything memorable — these are your Linux credentials, separate from your Windows login.

1d — Update Ubuntu packages
sudo apt update && sudo apt upgrade -y
1e — Install Python and pip
sudo apt install -y python3 python3-pip python3-venv
Accessing WSL2 later
Search "Ubuntu" in the Start menu, or open Windows Terminal and click the dropdown arrow to select Ubuntu. Your Windows files are accessible inside WSL2 at /mnt/c/Users/YourName/.

Step 2 — Annotate Race Footage in Roboflow

Roboflow handles frame extraction from video and browser-based annotation. You need roughly 300–500 annotated frames.

2a — Create a Roboflow project
  1. Go to roboflow.com and sign up for a free account.
  2. Click Create New Project.
  3. Name it racing-robot-detector.
  4. Set Project Type to Object Detection.
  5. Click Create Public Project.
2b — Upload your race footage
  1. Click Upload Data.
  2. Drag your .mp4 file onto the upload area.
  3. Roboflow asks how many frames to sample — choose 1 frame every 2 seconds. This gives ~225 frames from 7.5 minutes, which is a good starting point.
  4. Click Save and Continue.
2c — Auto-label with Roboflow's AI assist
  1. In the annotation view, click Assign Images → assign to yourself.
  2. Open the first image. Click the magic wand / AI Assist button in the toolbar.
  3. Use the Grounding DINO or Auto Label option. Type robot as the prompt.
  4. It will attempt to draw boxes automatically. Accept correct ones, delete wrong ones, and draw any it missed.

AI Assist won't be perfect — robots are unusual objects — but it speeds up the work considerably.

2d — Annotation tips
  • Label every robot visible in the frame, including partially obscured ones.
  • Draw boxes tightly around the robot body — not the dust cloud or shadow.
  • Leave frames where no robots are visible unannotated (not deleted) — they become negative training examples.
  • Include robots at close range, mid-range, and far distance.
2e — Generate a dataset version and export
  1. Once annotation is complete, click Generate New Version.
  2. Apply these augmentations: Flip Horizontal, Brightness ±15%, Blur up to 1.5px.
  3. Click Generate.
  4. When the version is ready, click Export Dataset → Format: YOLOv8 → Show Download Code.
  5. Copy the Python snippet — you will paste it into Google Colab in the next step.
Annotation quality over quantity
300 well-annotated frames will outperform 1000 carelessly labelled ones. Take time to be accurate, especially on small/distant robots.

Step 3 — Train YOLOv8n on Google Colab

Google Colab provides a free NVIDIA GPU. Training YOLOv8n on this dataset takes approximately 20–40 minutes.

3a — Open a new Colab notebook
  1. Go to colab.research.google.com and sign in.
  2. Click File → New Notebook.
  3. Click Runtime → Change runtime type → T4 GPU → Save.
3b — Install dependencies

In the first code cell, paste and run:

!pip install ultralytics roboflow --quiet
3c — Download your dataset from Roboflow

Paste the export snippet you copied from Roboflow in Step 2e. It will look like:

from roboflow import Roboflow
rf = Roboflow(api_key="YOUR_API_KEY")
project = rf.workspace("YOUR_WORKSPACE").project("racing-robot-detector")
version = project.version(1)
dataset = version.download("yolov8")

Note the path printed at the end — it will be something like /content/racing-robot-detector-1/.

3d — Train the model
from ultralytics import YOLO

model = YOLO('yolov8n.pt')   # nano — fast and efficient for the Pi

results = model.train(
    data='/content/racing-robot-detector-1/data.yaml',
    epochs=100,
    imgsz=640,
    batch=16,
    name='robot_detector',
    patience=20,              # stop early if not improving
)

Training progress is printed each epoch. Watch the mAP50 column — a value above 0.70 is a solid result for this task.

3e — Download the trained weights
from google.colab import files
files.download('/content/runs/detect/robot_detector/weights/best.pt')

This downloads best.pt to your Windows Downloads folder.

If mAP is below 0.50
Go back to Roboflow and annotate more frames, especially at the distances and angles where detections are weakest. More training data almost always beats longer training.

Step 4 — Install the Hailo Data Flow Compiler in WSL2

The Hailo DFC converts the PyTorch weights to a .hef file that runs on the Hailo-10H. It must run on x86-64 Linux — WSL2 provides this on Windows.

4a — Download the DFC from Hailo's developer zone
  1. Go to hailo.ai/developer-zone/ and log in (free registration).
  2. Navigate to Software Downloads → Hailo Software Suite.
  3. Download the Hailo Dataflow Compiler wheel for Ubuntu 22.04 / 24.04, x86-64, version 5.x.x (matching the 5.1.1 runtime on the Pi).
  4. The file will be named something like: hailo_dataflow_compiler-5.x.x-py3-none-linux_x86_64.whl
4b — Copy the wheel file into WSL2

In Ubuntu (WSL2), your Windows Downloads folder is accessible at:

ls /mnt/c/Users/YOUR_WINDOWS_USERNAME/Downloads/

Copy the wheel to your home directory:

cp /mnt/c/Users/YOUR_WINDOWS_USERNAME/Downloads/hailo_dataflow_compiler-*.whl ~/
4c — Create a Python virtual environment and install
cd ~
python3 -m venv hailo-env
source hailo-env/bin/activate

pip install hailo_dataflow_compiler-*.whl
pip install ultralytics

Installation takes a few minutes. You will see it pull in dependencies including TensorFlow and ONNX.

4d — Verify the installation
hailo --version

You should see output like hailo 5.x.x. If the command is not found, make sure the virtual environment is active (source ~/hailo-env/bin/activate).

Step 5 — Compile the Model to HEF

The DFC quantises the model to INT8 and compiles it for the Hailo-10H architecture. This produces the .hef file that runs on the AI HAT+ 2.

5a — Copy best.pt into WSL2
cp /mnt/c/Users/YOUR_WINDOWS_USERNAME/Downloads/best.pt ~/
5b — Export to ONNX

Make sure the virtual environment is active, then:

source ~/hailo-env/bin/activate

python3 - <<'EOF'
from ultralytics import YOLO
model = YOLO('best.pt')
model.export(format='onnx', imgsz=640, opset=11, simplify=True)
print("Exported to best.onnx")
EOF

This produces best.onnx in the same directory.

5c — Extract calibration images

The DFC needs a small set of real images to calibrate quantisation. Copy ~100 frames from your annotated dataset:

mkdir ~/calib_images
cp /mnt/c/Users/YOUR_WINDOWS_USERNAME/Downloads/racing-robot-detector-1/valid/images/*.jpg ~/calib_images/ 2>/dev/null || true
cp /mnt/c/Users/YOUR_WINDOWS_USERNAME/Downloads/racing-robot-detector-1/train/images/*.jpg ~/calib_images/ 2>/dev/null || true

# Keep only 100 images
ls ~/calib_images/ | tail -n +101 | xargs -I{} rm ~/calib_images/{}

If you exported the Roboflow dataset as a zip, extract it to your Windows Downloads folder first, then the paths above will work.

5d — Compile to HEF
hailo optimize best.onnx \
    --hw-arch hailo10h \
    --calib-path ~/calib_images/ \
    --output-har best_optimized.har

hailo compile best_optimized.har \
    --hw-arch hailo10h \
    --output-dir ~/hef_output/

The compiler runs for 5–15 minutes. On success you will find best.hef in ~/hef_output/.

5e — Copy the HEF to Windows

The WSL2 filesystem is accessible from Windows Explorer at:

\\wsl$\Ubuntu\home\YOUR_LINUX_USERNAME\hef_output\

Copy best.hef to somewhere convenient on Windows, such as your Desktop.

Architecture flag is critical
Always pass --hw-arch hailo10h. Using hailo8l (the older AI HAT+) will produce a file that fails to load on the AI HAT+ 2.

Step 6 — Deploy to the Raspberry Pi

6a — Copy the HEF to the Pi

From a Windows PowerShell or Command Prompt:

scp C:\Users\YOUR_WINDOWS_USERNAME\Desktop\best.hef pi@raspberrypi.local:~/Code/HackyRacingRobot/models/robot_detector.hef

Replace raspberrypi.local with your Pi's IP address if needed.

6b — Verify the HEF loads on the Pi

SSH into the Pi and run:

cd ~/Code/HackyRacingRobot
python3 - <<'EOF'
from hailo_platform import HEF
hef = HEF('models/robot_detector.hef')
info = hef.get_input_vstream_infos()[0]
print(f"Model loaded OK — input shape: {info.shape}")
EOF

Expected output: Model loaded OK — input shape: (640, 640, 3) (or similar).

6c — Update robot.ini

Add the following section to robot.ini:

[robot_detector]
enabled = true
model   = models/robot_detector.hef
conf    = 0.45          ; detection confidence threshold (0–1)
iou     = 0.45          ; NMS IoU threshold
Next step — wiring the detector into the robot
The robot detector module (robot/robot_detector.py) reads this config and is called from the camera thread in robot_daemon.py, alongside the existing ArUco detector. Detected robot bounding boxes appear in RobotState and the control loop can use them for collision avoidance.

Troubleshooting

ProblemLikely causeFix
wsl --install fails Virtualisation not enabled in BIOS Restart → BIOS setup → enable Intel VT-x or AMD-V
Colab disconnects mid-training Colab idle timeout Keep the browser tab active; add !ls cells to run periodically
mAP stays below 0.40 after 100 epochs Too few annotated frames or inconsistent labels Add 200+ more frames, review annotation quality, re-train
hailo optimize raises shape error ONNX opset mismatch Re-export with opset=11 and simplify=True
HEF fails to load on Pi Wrong --hw-arch flag Recompile with --hw-arch hailo10h
No detections at runtime Confidence threshold too high Lower conf in robot.ini to 0.30
Too many false detections Confidence threshold too low Raise conf to 0.55–0.65; add more negative frames to training
